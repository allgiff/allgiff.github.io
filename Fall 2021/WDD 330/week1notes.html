Week 1 Notes

Javascript Practice Exercises
  1. IO: input - programmer enters (or the computer draws from another location) output - Computer shares with the user.
  2. Key-Value Pair: consists of 2 related elements, a key (gender, color, etc.) and a value or variable, which belongs in the data set (female, blue, etc.)
  3. Equal comparison operator: (==) Compare is two things are equal
  4. Equal assignment operator: (=) Put something in a variable
  5. Modulus: returns the remainder of a division sum. If you need to check if something is even, use % == 2 or is the remainder divisible by two.
  6. Designing for mobile first before expanding to large view is going to be much simpler than the other way around.
    • Mobile traffic raised over 18% in a year (2013-2014).
    • Mobile first provided best experience across many devices
    • Future friendly
    • Addresses bandwidth constraints.
    • Start small and scale up
    • Provide users with want they want as quickly as possible.
    • Make it easy to reach key items without shifting their hand
  7. Quick loading promotes traffic and user engagement
    • Reduce image usage
    • Svg/webfonts are lower bandwidth
    • Optimize
    • GZIP files to eliminate unnecessary data
  8. Simplicity is key
  9. Addressing reachability issues can create a perception of increased app/site speed.
  10. Forms
    • Remove extra labels and display them in the field to save space.
    • Float labels make forms more user friendly
    • Reduce fields to promote user traffic
    • If a large form is necessary use sections or a progress bar.
    • Enable autocomplete
  11. Use the keyboard that relates to what the user is entering (if entering numbers, use number pad)
