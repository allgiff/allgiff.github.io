Practice Exercises
  • Objects: Represent entities in the real world (users, orders, etc.)
  • User actions (login, shopping cart) are represented by functions in properties in JavaScript
  • Method: Functions that are stored in object properties
    1. Methods are simplified, a form of shorthand for functions
  • this.: refers to the 'owner' of the method
    1. Arrow functions don't have a this. When used in the arrow funnction it is taken from outside
    2. Value changes depending on how a function is called
  • Object: self-contained set of related values and functions
  • If a property’s value is a function, it is known as amethod
  • Object Literal: object that is created directly in the language by wrapping all its properties and methods in curly braces
    1. can be passed as a parameter to a function
  • Properties of an object can be accessed using the dot notation
  • Can also be accessed using bracket notation. Only way to access nonstandard property and method names
  • Computed Properties: code can be placed inside square brackets and the property key will be the return value of that code
  • Call a method using dot or bracket notation. Same as invoking a function (needs parentheses)
  • in: Can be used to check wether an object has a certain property
  • hasOwnProperty(): Can be used to check whether an object has a property that is its own, rather than one that has been inherited from another object
  • for in: loop through all of an object’s properties and methods
  • Object.keys(): Will return an array of all the keys of any object that is provided as an argument
  • bject.entries(): Returns an array of key-value pairs
  • Adding Properties: Assigning a value to the new property
  • Nested Objects: Object that contains other objects
  • This: Refers to the object that it is in
  • object literal pattern: Creates a namespace for groups of related functions
  • JSON is a portable data format that uses JavaScript object literals to exchange information
  • Javascript has many built in math methods
  • Date object: Contains information about dates and times
  • Getter Method: return information about the date object
  • Setter Method: used to change the value of the date held in a Date object
  • The Regex object can be used to create regular expressions.
  • DOM: (Document Object Model) represents an HTML document as a network of connected nodes
    1. The HTML tag is the root node. All the other parts of the document are child nodes
    2. DOM can be used in any programming language
    3. Document.body: returns the body element of a web page
    4. Document.images: returns a node list of all the images contained in the document
    5. Document.links: returns a node list of all the a elements and area elements that have an href attribute
    6. Document.anchors: returns a node list of all the a elements that have a name attribute
    7. Document.forms: returns a node list of all the forms in the document
    8. document.querySelector(): allows you to use CSS notation to find the first element in the document that matches a CSS selector provided as an argument
    9. document.querySelectorAll(): uses CSS notation but returns a node list ofallthe elements in the document that match the CSS query selector
    10. childNodes: list of all the nodes that are children of the node
    11. children: only returns any element nodes that are children of that node, so will ignore any text nodes
    12. parentNode: returns the parent node of an element
    13. nextSibling: returns the next adjacent node of the same parent
    14. previousSibling: returns the previous adjacent node
    15. getAttribute(): returns the value of the attribute provided as an argument
    16. setAttribute: change sthe value of an element’s attributes. Using two arguments: the attribute that you wish to change, and the new value of that attribute
    17. className: allows the class of an element to be set directly
    18. classList: list of all the classes an element has
    19. add: adds a class to an element without overwriting any classes that already exist
    20. remove: removes a specific class from an element
    21. toggle: adds a class if an element doesn’t have it already, and remove the class if it does have it
    22. contains: checks to see if an element has a particular class
    23. createElement(): takes a tag name as a parameter and returns that element
    24. document.createTextNode(): creates a text node using a parameter
    25. appendChild(): adds another node (given as an argument) as a child node
    26. nsertBefore(): places a new element before another element
    27. removeChild(): used to remove an element from a page
    28. replaceChild(): used to replace one node with another
    29. innerHTML: returns all the child elements of an element as a string of HTML
