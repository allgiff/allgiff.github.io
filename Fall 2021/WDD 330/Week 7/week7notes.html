  • Functions are: First-Class Objects: can be passed around in the same way as every other value.
  • Call Method: Used to set the value of thos inside a function to an object provided as the first argument
  • Description Property: Can be used to describe what a function does
  • Memoization: The result of a function can be stored in the cache so that it does not have to loop through many times
  • Immediately Invoked Function Expression: An anonymous function that is invoked as soon as it’s defined.
    • Temp: Temporary variable that only exists while an IIFE is invoked
    • Initialization Code: A form of IIFE that initalizes code
    • IIFE can be used to enclose a block of code inside its own private scope so it doesn’t interfere with any other part of the program (can be added or removed seperatly)
  • Self-defining functions: Done by assigning an anonymous function to a variable that has the same name as the function
  • Init-time branching: enables the functions to work more effectively in the browser, and avoid checking for features every time they’re invoked
  • Recursive function: invokes itself until a certain condition is met
  • Promise: represents the future result of an asynchronous operation
    • When a promise is created it is pending. Once an opperation is completed it is settled & can result in 2 different outcomes
      • Resolved ― the asynchronous operation was completed successfully
      • Rejected ― the asynchronous operation didn’t work as expected, wasn't successfully completed or resulted in an error
    • Creating a promise: Created using a constructor function
      • takes an executor as an argument, which initalizes and starts the opperation
      • Accepts two functions as arguments:
        • resolve(): called if opperation is successful
        • reject(): called if opperation fails
  • Closures: Powerful javascript feature
  • Generator: special functions used to produce iterators that maintain the state of a value
  • Pure Functions: return value only depends on given arguments, no side effects, will always return the same result when the same arguments are given
  • Higher-order functions: accept another function as an argument, or return another function as a result, or both
  • Currying: process that involves the partial application of functions
  • Ajax: technique that allows web pages to communicate asynchronously with a server. (updates pages without reloading, like a live server)
  • fetch(): returns a promise that resolves to the reponse returned from the URL
  • response interface: deals with the object returned when the promise is fulfilled
  • redirect(): used to redirect to another URL, and creates a new promise
  • text(): reads text from a response and returns a promise that can be treated as a string
  • blob(): used to read raw data and returns a promise that resolves with a blob object
  • Request objects: created using the request constructor, and gives control over the request
  • headers: used to pass additional information about a request or response
  • Ajax can also be used to send information, usually as a JSON string
  • Ajax is still subject to change because it is a 'living standard'
