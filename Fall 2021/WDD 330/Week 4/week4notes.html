  • Forms: allow users to enter certain information into a web page or app
    • You can use the mane to identify a form
    • Elements: method that returns an HTML collection of the elements contained in the form
    • form controls can also be accessed using name attributes
    • form.submit()method: submit the form automatically
    • form.reset()method: reset all the form controls back to their initial values specified in the HTML
    • form.action property: can be used to set theactionattribute of a form, so itâ€™s sent to a different URL to be processed on the server
    • focus event: occurs when an element is focused on
    • blur event: occurs when the user moves the focus away from the form element
    • change event: occurs when the user moves the focus away from the form element after changing it
    • value property: can be used to retrieve the text inside the field.
    • Input Type: Used to label and adjust form boxes for the type of input. (ex type="password" makes the password secret/invisible)
      • Check boxes: allow the user to select multipul options
      • Radio Button: similar to check boxes, but only allow the user to select one item
      • Hidden: input fields can be hidden. Used for settings or already added info
      • file: Used to upload files
    • Dtop down list: used to select one or more options from a list of values
    • Text area: used to enter long pieces of text (ie comment or blog post)
    • Required: requires the user to enter something in the required field before the form will submit
    • Validation: ensures all items have been completed (ie password is at least 8 characters, etc.)
  • Object Oriented programming: Used to model representations of objects in real world
    • Encapsulation: Inner workings are hidden inside the object, with only essenntial functions shown
    • Ploymorphism: The same process being used by different objects. Objects can share the same method
      • Different objects can have the same method, but use it in different ways
    • Inheritance: taking the features of one object, and adding new features
    • Constructor Function: defines the properties and methods of an object
    • Classes: can be declared in JavaScript, does the same thing as a constructor function, but is more similar to other programming languages.
    • Constructor Property: returns the constructor function that created it
    • Static Methods: called by the class directly rather than by class instances
    • Prototypal Inheritance: every class has a prototype property that is shared by every instance of the class.
      • Can be used to add extra properties and methods to the class after it has been created.
      • Prototypes should be used for adding new properties and methods after the class has been declared
    • Enumerable Properties: If I property is not enumerable it will not show up when a for-in loop is used
    • Extends: Causes a class to inherit from another class
    • Object.getOwnPropertyDescriptor(): gets and defines property descriptors
    • It is possible to create objects using other objects as a blueprint. Can be done using Object Create
    • Mixins: Used to add properties to an object, often a large amount all at once
      • Can also be used to create a copy function, which creates an exact copy of an object
  • Modular JavaScript
    • Module: self contained piece of code that provides functions and methods that can be used by other modules
    • Keeps code organized in seperate reuseable files, and makes it easier to maintain code
    • Coupling: Elements or modules that are dependant on each other
    • export: keyword that takes a module and applies part of it to the code you are working with
    • Default export: a single variable, function or class in a module that can be imported without having to be explicitly named (Do Not use more than once)
    • Alias: assigned alias in imported module does not have to match the name in the original module
    • Node modules use a slightly different notation than ES6
